# Password Security & Hash Analysis

## Project Overview

This project was completed as part of the Cyber Security & Ethical Hacking Internship. The objective of this project was to understand password security concepts, password storage mechanisms, hash functions, and the importance of strong password policies.

## Objective

* Understand password storage techniques.
* Learn different hash algorithms.
* Study password strength analysis.
* Compare weak and strong passwords.
* Understand the importance of secure authentication practices.

## Tools Used

* Kali Linux
* Nano Text Editor
* Linux Command Line

## Project Activities

### 1. Password Strength Analysis

Different sample passwords were analyzed and categorized based on their strength.

Examples:

| Password     | Strength |
| ------------ | -------- |
| password123  | Weak     |
| admin123     | Weak     |
| welcome123   | Weak     |
| Strong@2026# | Strong   |

### 2. Hashing Concepts

The following hashing algorithms were studied:

#### MD5

* Produces a 128-bit hash.
* Considered insecure for modern applications.

#### SHA-1

* Produces a 160-bit hash.
* More secure than MD5 but no longer recommended.

#### bcrypt

* Adaptive password hashing algorithm.
* Widely used for secure password storage.

### 3. Password Security Risks

Common risks associated with weak passwords:

* Easy to guess passwords.
* Reused passwords across multiple platforms.
* Short passwords with predictable patterns.
* Lack of multi-factor authentication.

### 4. Security Best Practices

Recommended password security measures:

* Use passwords with at least 12 characters.
* Include uppercase and lowercase letters.
* Use numbers and special characters.
* Enable Multi-Factor Authentication (MFA).
* Avoid password reuse.
* Use a password manager.
